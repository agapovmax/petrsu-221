### Расширение файла .S
.S (с большой буквы) означает, что файл должен быть препроцессирован. То есть перед ассемблированием компилятор выполнит в нем все директивы вроде #include, #define, #ifdef и т.д.

Файлы с расширением .s (маленькая буква) обрабатываются ассемблером напрямую, без препроцессора.



### Разница между %eax и %ebx
%eax часто используется для хранения возвращаемого значения функции.

%ebx в некоторых соглашениях (например, при системных вызовах в Linux) используется для передачи первого аргумента (например, файлового дескриптора stdout = 1 для вывода на экран).

%eax часто используется для активных вычислений, промежуточных результатов и временных значений.

%ebx, %esi, %edi часто используются для хранения важных указателей или значений, которые должны сохраняться на протяжении работы функции или цикла (они часто относятся к callee-saved регистрам).
